# JSX Preview

## 課題
コードの見た目を確認するためにブラウザに切り替えるのが面倒。

## ゴール
JSX要素にホバー → [JSX Preview] クリック → サイドパネルにプレビュー表示。
アプリ起動なしで動く。

## アプローチ（v2: React実行方式）

### 旧アプローチの問題
JSXをHTMLに変換してそのまま描画していた。
→ アイコンライブラリ、カスタムコンポーネント、CSS-in-JSが動かない。

### 新アプローチ
Webview内でReactを実行し、実際のアプリと同じ方法で描画する。

```
JSX要素をクリック
  ↓
extractJsxBlock: JSXブロックを取得
  ↓
esbuild: JSXブロック + importを解決してバンドル
  ↓
Webview: React + バンドルされたJSを実行して描画
```

### これにより対応できるもの
- アイコンライブラリ（react-icons等）
- カスタムコンポーネント
- styled-components / Emotion
- Tailwind CSS（プロジェクトの設定をそのまま利用）

### 課題
- Propsやstateに依存するコンポーネントの扱い
- importの解決パスをプロジェクトに合わせる必要がある
- esbuildバンドルの速度

## 将来の改善案
- ホバーのみでプレビュー表示（クリック不要）
- 表示方法をオプションで切り替え可能にする
- JSX解析精度の向上（正規表現 → パーサー）
- プレビューサイズ切り替え（モバイル / デスクトップ）
